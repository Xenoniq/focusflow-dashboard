# FocusFlow Command Center

**FocusFlow Command Center** — интерактивный дашборд для управления продуктивностью. Проект демонстрирует создание, добавление, удаление и взаимодействие UI-компонентов с использованием ООП, ES6 Modules, асинхронного JavaScript и сторонних API.

## Тематика проекта

Тема дашборда — персональная панель продуктивности. Пользователь может работать с задачами, цитатами, погодой, валютными курсами и таймером фокуса в одном интерфейсе.

## Реализованные виджеты

1. **ToDoWidget**  
   Список задач с добавлением, удалением и отметкой выполненных задач.

2. **QuoteWidget**  
   Виджет случайной цитаты. Цитаты хранятся в массиве внутри класса, при нажатии кнопки цитата обновляется.

3. **WeatherWidget**  
   API-виджет погоды. Получает данные из Open-Meteo API.

4. **CurrencyWidget**  
   API-виджет валют. Получает актуальные курсы из Open ExchangeRate API.

5. **FocusTimerWidget**  
   Таймер фокус-сессии с запуском, паузой и сбросом.

## Работа с API, JSON и асинхронностью

В проекте реализованы два виджета, которые получают данные со сторонних REST API:

### WeatherWidget

Использует:

- Open-Meteo Geocoding API для поиска города и получения координат;
- Open-Meteo Forecast API для получения текущей погоды.

В коде используется:

- `fetch()` для HTTP-запросов;
- `async/await` для асинхронной логики;
- `response.ok` для проверки успешности ответа;
- `response.json()` для преобразования JSON-ответа в JavaScript-объект;
- `try/catch` для обработки ошибок;
- `AbortController` для отмены предыдущего запроса при новом запросе или удалении виджета.

### CurrencyWidget

Использует Open ExchangeRate API для получения актуальных курсов валют.

В коде используется:

- `fetch()`;
- `async/await`;
- `response.ok`;
- `response.json()`;
- `try/catch`;
- `AbortController`.

Выбранные API не требуют API-ключей, поэтому проект можно безопасно запускать локально и публиковать через GitHub Pages без хранения приватных токенов в коде.

## ООП-архитектура

Проект построен на классах и разделён на модули.

### `UIComponent`

Базовый абстрактный класс для всех виджетов.

Отвечает за:

- создание общей DOM-обёртки виджета;
- общий заголовок и кнопки управления;
- сворачивание виджета;
- закрытие виджета;
- регистрацию обработчиков событий;
- корректное удаление обработчиков через `destroy()`;
- вывод сообщений и состояния загрузки.

### Конкретные классы виджетов

Классы `ToDoWidget`, `QuoteWidget`, `WeatherWidget`, `CurrencyWidget` и `FocusTimerWidget` наследуются от `UIComponent` и переопределяют метод `renderContent()`.

Каждый виджет хранит собственное состояние внутри экземпляра класса. Например:

- задачи хранятся в свойстве `this.tasks` внутри `ToDoWidget`;
- текущие данные погоды хранятся в `this.weatherData` внутри `WeatherWidget`;
- валютные курсы хранятся в `this.ratesData` внутри `CurrencyWidget`.

### `Dashboard`

Класс, управляющий всей панелью.

Отвечает за:

- регистрацию типов виджетов;
- добавление виджетов через `addWidget(widgetType)`;
- удаление виджетов через `removeWidget(widgetId)`;
- хранение коллекции активных виджетов в `Map`;
- обновление счётчика активных компонентов;
- отображение пустого состояния панели;
- защиту от повторного добавления одинакового типа виджета;
- вывод уведомлений о действиях пользователя.
